The Settings page provides both control and configuration of your Stratux device.

Use the toggles in the Hardware section to control which hardware devices are active.

NOTE: Only hardware enabled here will appear on the Status page.

  • 978 MHz and 1090 MHz enable reception of traffic and weather on those respective frequencies. You must have two receivers installed to receive both frequencies.
  • GPS enables use of a connected GPS on GPIO or USB interfaces. Devices (u-blox or MTK3339 chipset) connected on GPIO UART should be set to 9600 baud for autoconfiguration.
  • AHRS enables a MPU6050 6-axis gyro and BMP180 pressure sensor connected to the GPIO I2C pins. Use in navigation EFB software additionally requires an AHRS Output option to be selected as described below.

The Diagnostics section helps with debugging and communicating with the Stratux project contributors via GitHub, Slack, and the reddit subgroup.

  • Traffic Markings prepends two letters in front of the callsign of traffic targets within your navigation application. The first letter will be u for traffic received on 978/UAT, and e for 1090-ES. The second letter indicates the type of target: a for ADS-B air-to-air, r for ADS-R ground-to-air rebroadcast, and t for TIS-B ground-to-air broadcast.
  • Verbose Logs logs additional messages to stratux.log to aid in development and troubleshooting. This file is accessible from the Logs menu available on the left.
  • Record Logs enables logging to a series of files for your Stratux device including raw data recorded for UAT traffic and weather, 1090 traffic, GPS messages, and AHRS messages. The log files are accessible from the Logs menu available on the left.

The Configuration section lets you adjust the default operation of your Stratux device.

  • Enabling AHRS Output to EFB will send AHRS data from a hardware AHRS (enabled in "Hardware") or GPS Simulated AHRS (enabled in "Experimental Configuration Options") to attached compatible clients over the WiFi interface using the GDL90 protocol. This option must be disabled for compatibility with ForeFlight 7.6.
  • To avoid having your own aircraft appear as traffic and scare the bejeezus out of you, you may provide your Mode S code. This will also give you the option of detecting your airplane's pressure altitude, if it is Mode S or ADS-B equipped. You can find this value in the FAA N-Number Registry for your aircraft. You should use the hexadecimal value (not the octal value) for this setting. No validation is done so please ensure you enter your valide Mode S value.
  • The Weather page uses a user-defined Watch List to filter the large volume of ADS-B weather messages for display. Define a list of identifiers (airport, VOR, etc) separated by a spaces. For example KBOS EEN LAH LKP. You may change this list at any time and the Weather page will start watching for the updated list immediately.
    NOTE: To save your changes, you must either tap somehwere else on the page or hit ENTER or RETURN or GO (or whatever your keyboard indicates).
  • The SDR (software defined radio) receiver support an adjustment in the form of a PPM Correction. From the Raspberry Pi, you may use the command kal -g 48 -s GSM850 to scan for available channels in your area. Then use the command kal -g 48 -c channel# to calculate the PPM.
    NOTE: You will need to perform all commands as root by issuing the command: sudo su -. You will need to stop the Stratux software before running the calibration process. You can stop all of the Stratux processes with the command: pkill screen.

The Commands section lets you upgrade the system software, or safely shutdown / reboot your Stratux device.

  • Stratux software updates can be loaded if you Click to select System Update file and follow the prompts. Upgrade packages will be available on stratux.me or GitHub.
  • Shutdown will immediately shutdown the Stratux. You may then safely remove power.
  • Reboot will immediately reboot the Stratux. After the reboot you may have to rejoin the WiFi connection to reconnect.

The Experimental Configuration Options section contains new alpha / beta features that enhance Stratux but need validation before they are considered production ready.

  • ForeFlight 'Flight Simulator' AHRS Output uses a ForeFlight's flight simulator interface to send position, traffic, and AHRS data. It is intended to be a developer tool for evaluating AHRS functions, and may not work correctly in the air. It is also incompatible with the normal Stratux operating mode, and will cause FIS-B weather to be disabled on ForeFlight, and all Stratux output to be disabled for other apps.
  • Traffic Demo circulates a 40-mile diameter flock of simulated traffic around your position. This can be used to validate that your navigation / EFB software is connecting and displaying traffic.
  • FLARM-Formatted Serial Traffic Output sends traffic formatted with the FLARM glider computer protocol out over a USB serial connection (/dev/ttyUSB0, 38400 bps). It will disable any USB serial devices that appear as /dev/ttyUSB0 (e.g. BU-353), but not those that appear as ttyACM0 (e.g VK-172)
  • GPS Simulated AHRS tracks GPS groundspeed, vertical speed, and heading, and uses a dynamic model to calculate the pitch and roll a coordinated airplane would experience. Think of it as an intelligent partial-panel. GPS must be enabled and AHRS disabled in the Hardware section for this to function.
  • Use ADS-B Ownship Altitude and Callsign will attempt to detect your aircraft's Mode S or ADS-B output if its Mode S code is entered in the Configuration section. If detected, it will use the callsign and pressure altitude when sending ownship messages to your navigation software.
